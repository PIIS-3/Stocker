🧪 Guía y Estrategia de Testing (TDD)

Documento de planificación sobre cómo aseguraremos la calidad del software (Próxima Fase de Desarrollo).

Backend (Pytest + FastAPI.TestClient)

Gracias a la separación de la Lógica CRUD, probar el backend será extremadamente modular.

1. Pruebas Unitarias (CRUD)

Herramienta: pytest

Se crearán funciones test_create_product() en la carpeta backend/tests/crud/. Mockearemos (fingiremos) la Sesión de Base de Datos para probar la lógica sin afectar al disco duro.

2. Pruebas de Integración (API Endpoints)

Herramienta: TestClient de FastAPI

Lanzaremos peticiones HTTP falsas a /api/products y comprobaremos si nos devuelve Pydantic Validation Errors (Error 422) al enviar JSON corruptos o códigos 200 cuando es correcto.

Frontend (Vitest + React Testing Library)

Vite ya incluye el entorno nativo de Vitest (mucho más rápido que el antiguo Jest).

1. Unit Tests de Componentes Visuales

Comprobaremos que componentes aislados (como el Navbar) renderizan correctamente si le inyectamos Props como "Usuario Logeado".

2. Mocks de Interceptores Axios

Simularemos respuestas de red en la capa services/ para comprobar cómo reaccionan las pantallas a un Error de Red (500 Server Error) de manera segura.
