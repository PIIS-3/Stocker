⚛️ Metodología: Diseño Atómico

Entiende cómo estructuramos nuestra interfaz para que sea modular, reutilizable y fácil de escalar.

¿Qué es el Diseño Atómico?

Es una metodología que divide la interfaz en componentes básicos que se combinan para formar estructuras más complejas. Esto evita duplicidad de código y garantiza consistencia visual.

⚛️ Átomos

Las piezas más pequeñas: botones, etiquetas, inputs o iconos. No funcionan solos, son los ladrillos de la UI.

🧪 Moléculas

Grupos de átomos unidos. Por ejemplo, un campo de búsqueda (Input + Botón). Tienen una función específica.

🦠 Organismos

Secciones complejas formadas por moléculas y átomos. Ejemplo: Un Navbar o un Formulario completo.

🖼️ Templates

Esquemas de página que definen el layout (donde va el sidebar, el header, etc.) sin contenido real.

Reglas de Oro del Proyecto

  • Jerarquía Estricta: Un Átomo no puede contener una Molécula. Un Organismo puede contener cualquier cosa por debajo de él.
  • Estilos Centralizados: Los átomos deben usar variables de CSS o clases de Tailwind definidas en el `tailwind.config`.
  • Prop Drilling: Evita pasar props a través de demasiados niveles. Si un Organismo necesita datos complejos, usa Contexto o inyecta el servicio.
  • Exportación: Cada carpeta (atoms, molecules, etc.) tiene un `index.ts` que exporta todos sus componentes para facilitar los imports.

Ejemplo Práctico

Imagina la página de Tiendas:

  1. Átomo: El botón de "Añadir Tienda".
  2. Molécula: El `SearchInput` de la parte superior.
  3. Organismo: El `StoreForm` (Modal + Inputs + Botones).
  4. Template: El `AdminLayout` (Sidebar + Contenido principal).
  5. Página: `StoresList.tsx` que une todo y le da datos.
