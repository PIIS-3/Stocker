⚙️ Guía Paso a Paso: CRUD Backend

Tutorial sobre cómo añadir de forma correcta, modular y escalable un nuevo recurso ("entidad") en el servidor FastAPI.

En este ejemplo, utilizaremos la entidad Store (Tiendas) como el estándar de oro del proyecto.

Copia este bloque de texto, sustituye [NOMBRE_ENTIDAD] y envíaselo al chat:

Implementa un nuevo CRUD completo para la entidad "[NOMBRE_ENTIDAD]".

Utiliza los archivos de la entidad "Store" como tu estándar arquitectónico estricto:
- Modelos: backend/app/models/store.py (Usa Base, Create, Update, Response y el Modelo ORM puro).
- CRUD Pura: backend/app/crud/stores.py
- Endpoints: backend/app/api/endpoints/stores.py

REGLAS OBLIGATORIAS:
1. El modelo ORM de BD debe heredar de `TimestampMixin` (backend/app/models/mixins.py) para autogestionar created_at y updated_at.
2. Añade el nuevo modelo a backend/app/models/__init__.py
3. En el archivo CRUD, devuelve solo datos u objetos ORM (o None). NO importes ni lances HTTPException aquí.
4. En el archivo de endpoints, inyecta `Depends(get_db)`. Valida conflictos (409) y no encontrados (404).
5. Los endpoints deben replicar los comentarios visuales de separación (# ── GET /entidad/ ──).
6. Crea siempre un endpoint adicional `GET /by-name/{name}` antes del `GET /{id}`.
7. Registra el router en backend/app/api/router.py.
8. Todo el código debe estar excelentemente documentado en español.

Fase 1: Base de Datos y Modelos

Paso 1.1 Crear Modelo SQLModel

Archivo: backend/app/models/store.py

from typing import Optional
from datetime import datetime
from sqlmodel import SQLModel, Field
from .mixins import TimestampMixin

class StoreBase(SQLModel):
    name: str = Field(unique=True, index=True, min_length=1)
    address: str

class StoreCreate(StoreBase):
    pass

class StoreUpdate(SQLModel):
    name: Optional[str] = Field(default=None, min_length=1)
    address: Optional[str] = None

class Store(TimestampMixin, StoreBase, table=True):
    id_store: Optional[int] = Field(default=None, primary_key=True)

class StoreResponse(StoreBase):
    id_store: int
    created_at: Optional[datetime] = None
    updated_at: Optional[datetime] = None

Paso 1.2 Registrar en el Índice

Archivo: backend/app/models/__init__.py

from .store import Store, StoreBase, StoreCreate, StoreResponse

Fase 2: Lógica y Rutas API

Paso 2.1 Crear CRUD (Lógica)

Archivo: backend/app/crud/stores.py

from sqlmodel import Session, select
from app.models.store import Store, StoreCreate, StoreUpdate

def get_stores(db: Session, skip: int = 0, limit: int = 100) -> list[Store]:
    return db.exec(select(Store).offset(skip).limit(limit)).all()

def create_store(db: Session, store_in: StoreCreate) -> Store:
    db_store = Store.model_validate(store_in)
    db.add(db_store)
    db.commit()
    db.refresh(db_store)
    return db_store

Paso 2.2 Publicar Endpoints

Archivo: backend/app/api/endpoints/stores.py

from fastapi import APIRouter, Depends, HTTPException, status
from app.database import get_db
from app.crud import stores as crud

router = APIRouter()

@router.get("/", response_model=list[StoreResponse])
def read_stores(db: Session = Depends(get_db)):
    return crud.get_stores(db)
