🔊 Audio y Notificaciones

Documentación sobre cómo funciona el sistema de sonidos, las preferencias del usuario y la gestión del desbloqueo de audio en el navegador.

🚀 Funcionamiento General

El sistema de audio está diseñado para cumplir con las políticas de Autoplay de los navegadores modernos, que impiden que una página emita sonidos sin una interacción previa del usuario.

1. Desbloqueo de Audio

Para que los sonidos de notificación funcionen, el usuario debe realizar una acción (clic). Hemos centralizado esto en el componente SoundToggle ubicado en el Dashboard.

  • Al activar el sonido, se ejecuta un "cebado" silencioso usando tanto HTMLAudio como WebAudio API.
  • Esto marca la pestaña como "autorizada" para reproducir audio en el futuro.

2. Preferencias Globales

La preferencia del usuario se guarda en localStorage bajo la clave notifySoundEnabled.

  • isSoundEnabled(): Consulta si el usuario quiere escuchar sonidos.
  • setSoundEnabled(value): Guarda la nueva preferencia.

🛠️ Utilidades Técnicas

Todas las funciones residen en frontend/src/utils/audio.ts:

// Reproducir un sonido desde URL (Ej: Notificaciones)
await playSoundFromUrl('/sounds/notification-chime.mp3');

// Reproducir un tono sintetizado (Fallback si el archivo falla)
playTone('success'); // 'success' | 'error' | 'info'

🔔 Notificaciones (Toasts)

El componente ToastNotification integra automáticamente el sonido:

<ToastNotification
  variant="success"
  title="Tienda Guardada"
  message="Los cambios se aplicaron con éxito"
  playSound={true}
  soundUrl="/sounds/notification-chime.mp3"
/>

Nota: Aunque playSound sea true, el componente verificará internamente isSoundEnabled() antes de intentar reproducir nada.
