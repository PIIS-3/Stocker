🛡️ Guía de Control de Acceso (RBAC)

Políticas de seguridad, jerarquía de roles y restricciones de módulos en Stocker.

Jerarquía de Roles

El sistema implementa una estructura de tres niveles para garantizar la integridad de los datos y la seguridad de la plataforma.

👑 SuperAdmin (Administrador Global)

Acceso total y sin restricciones. Es el único rol capaz de gestionar tiendas físicas y eliminar usuarios/empleados.

👔 Manager (Director de Tienda)

Enfocado en la gestión operativa de su inventario. Puede gestionar productos y categorías, pero no tiene acceso a la configuración de tiendas ni a la gestión global de usuarios.

🛠️ Staff (Personal Operativo)

Rol puramente transaccional. Puede ver, crear y editar productos o categorías, pero tiene prohibido borrar cualquier recurso del sistema.

Implementación Técnica

Backend (FastAPI)

La seguridad se aplica mediante dependencias en los endpoints:

  • get_current_superadmin: Valida que el rol sea exactamente SuperAdmin.
  • get_current_admin: Permite SuperAdmin y Manager.
  • get_current_employee: Valida que el usuario esté activo en la base de datos.

Frontend (React)

El sistema utiliza dos mecanismos complementarios:

  • Protección de Rutas: El componente RoleProtectedRoute bloquea el acceso por URL y redirige al dashboard si el rol no es válido.
  • UI Condicional: El servicio authService.can(action, module) oculta dinámicamente botones de borrado o elementos del sidebar.
