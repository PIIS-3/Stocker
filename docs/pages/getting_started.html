🚀 Guía de Operaciones (Getting Started)

Aprende a levantar el proyecto, gestionar la base de datos y validar tu código usando el Makefile central.

Requisito: Debes tener Docker Desktop instalado y abierto en tu ordenador antes de empezar. Todos los comandos se ejecutan desde la raíz del proyecto Stocker/.

1. Encender el Proyecto

No necesitas configurar variables manualmente. El comando up se encarga de crear el .env inicial por ti: 

make up

Una vez completado (puede tardar unos minutos la primera vez), accede a:

  • Frontend React: http://localhost:5173
  • Backend API: http://localhost:8000 (Swagger en /docs)

2. Poblado de Datos (Seed)

Para tener algo que ver en la interfaz, inyecta los datos de prueba iniciales: 

make seed

Este comando es idempotente: puedes ejecutarlo varias veces y no duplicará los datos existentes.

3. Cambios en la Base de Datos (Alembic)

Si modificas un modelo en backend/app/models/, sigue este flujo:

  1. Crear migración:
    make migration m="descripcion del cambio"
  2. Aplicar cambios:
    make migrate

Esto asegura que tu base de datos en Docker esté siempre sincronizada con tu código Python.

4. Herramientas Útiles

  • make up o Iniciar: levanta el entorno Docker completo (con build).
  • make down o Apagar todo: apaga los contenedores del proyecto (Libera memoria y CPU de tu ordenador).
  • make restart o Limpieza total: reinicia el stack completo. Si algo falla catastróficamente, úsalo.
  • Ver Logs: make logs (Útil para debugear errores del servidor o la base de datos).
  • make seed: ejecuta el seeder Python idempotente.
  • make seed-sql: ejecuta el seed SQL legacy.
  • make test: **Global:** Ejecuta todos los tests (Backend + Frontend).
  • make test-backend: ejecuta solo los tests del backend (pytest).
  • make test-frontend: ejecuta solo los tests del frontend (Vitest).
  • make lint: **Global:** Ejecuta todos los linters y validaciones.
  • make format-frontend: aplica formato Prettier al frontend.

5. Antes de subir tu código (PR Check)

Es obligatorio que tu código pase todas las validaciones antes de enviarlo a revisión: 

make pr-check

Si este comando falla, el equipo no aceptará tu Pull Request.
