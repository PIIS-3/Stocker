🚀 Guía de Operaciones (Getting Started)

Aprende a levantar el proyecto, apagarlo, limpiarlo e inyectar datos falsos.

Aviso Importante: La terminal debe estar abierta en la carpeta /docker o debes añadir cd docker antes de ejecutar estos comandos o fallarán.

1. Preparar y Encender el Proyecto

Primero, crea un archivo .env copiando el de ejemplo dentro del directorio /docker:

cp .env.example .env

Luego, este comando construye las imágenes (Frontend en Node 25, Backend en Python 3.12 y Database en Postgres 16) y lanza los tres contenedores en segundo plano -d.

docker compose up --build -d

Una vez completado, las herramientas estarán en:

  • Frontend React: http://localhost:5173
  • Backend API: http://localhost:8000 (Documentación interactiva en /docs)

2. Inyectar datos de prueba (Seed Database)

Como Postgres arranca en limpio, FastAPI usará Alembic para crear las tablas vacías. Una vez estén creadas (10 segundos tras encender el servidor), ejecuta el seeder del backend para cargar categorías y productos de prueba de forma idempotente.

make seed

Equivalente directo sin Makefile:

docker exec stocker_api python -m app.seed

El script está en backend/app/seed.py, utiliza SQLModel Session y crea/actualiza registros sin duplicarlos (por category_name y sku).

Alternativa legacy con SQL plano:

make seed-sql

3. Realizar cambios en la Base de Datos (Migraciones)

Si alteras algún archivo dentro del paquete models/ de Python para añadir o quitar columnas, debes ordenarle a Alembic que compare el modelo nuevo con el antiguo de Postgres y genere el script de actualización.

  1. Genera la captura autogenerada:
    docker exec stocker_api alembic revision --autogenerate -m "He añadido una columna de Stock"
  2. Aplica la actualización a la base de datos real:
    docker exec stocker_api alembic upgrade head

4. Apagar y Limpiar

Si quieres apagarlo todo temporalmente para conservar recursos y disco:

docker compose down

Si notas fallos extraños y quieres **borrar todo el progreso (Resetear de Fábrica)**, eliminando la base de datos, los datos guardados en el disco virtual de Docker y las imágenes construidas, usa:

docker compose down -v --rmi all --remove-orphans

5. Comandos de Makefile para desarrollo

Para estandarizar el flujo del equipo, ejecuta estos comandos desde la raíz del repositorio:

  • make up: levanta el entorno Docker completo (con build).
  • make down: apaga los contenedores del proyecto.
  • make restart: reinicia el stack completo.
  • make seed: ejecuta el seeder Python idempotente.
  • make seed-sql: ejecuta el seed SQL legacy.

6. Parámetros de entorno del equipo

Estas variables viven en docker/.env (base en docker/.env.example):

  • POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB: credenciales y nombre de DB.
  • API_PORT: puerto local del backend.
  • FRONTEND_PORT: puerto local del frontend.
  • FRONTEND_URL: origen permitido por CORS para FastAPI.
  • VITE_API_URL: URL base consumida por el frontend.
