Stocker - Arquitectura de Código

Guía de directorios, responsabilidades y librerías clave instaladas en el ecosistema.

Estructura General del Repositorio

/ ├── backend/ # Código fuente y configuraciones de la API Python ├── frontend/ # Código fuente de React (Interfaz de usuario) ├── docker/ # Archivos de orquestación (docker-compose.yml y envs) ├── docs/ # Manuales HTML (Stocker Documentation) ├── .editorconfig # Estilos universales para editores e IDEs del equipo └── README.md # Instrucciones de inicio rápido

1. Backend (API en FastAPI)

El backend sigue una arquitectura basada en capas (Rutas -> Schemas -> Operaciones -> Modelos) para separar claramente la recepción de datos de la escritura en base de datos.

backend/ ├── alembic/ # Contiene el historial de migraciones ('versions/') generado por Alembic ├── alembic.ini # Configuración de Alembic y conexión a DB ├── app/ │ ├── api/ # Endpoints URL. Incluye /endpoints/* y el router.py centralizado │ ├── crud/ # Funciones lógicas limpias de Base de Datos ('get_user', 'create_product') │ ├── models/ # El 'ADN' de tus tablas SQL. Defines Clases usando SQLModel (un archivo por entidad) │ ├── database.py # Creación de la sesión ('Session') hacia PostgreSQL (con pool_pre_ping) │ └── main.py # Punto de arranque de Uvicorn, engloba el Router centralizado y CORS └── requirements.txt # Listado absoluto de librerías para que Docker las instale

📚 Dependencias Clave (requirements.txt)

fastapi

Framework moderno de alto rendimiento para APIs. Genera la documentación interactiva (Swagger) sola.

uvicorn

El servidor web ASGI que expone la FastAPI al mundo y permite la recarga en vivo de tus cambios (-reload).

sqlmodel

Biblioteca moderna que fusiona Pydantic y SQLAlchemy. Permite definir validaciones de API y Modelos Base de Datos en la misma Clase en `models.py`, evitando repetir el código 2 veces.

alembic

Herramienta oficial de migraciones para SQLAlchemy. Vigila tus "models.py" y autogenera tablas nuevas sin perder tu información cuando el proyecto crece.

psycopg2-binary

El driver o "traductor" de bajo nivel que permite que Python y PostgreSQL hablen entre sí sin latencia.

pydantic

Capa de validación estricta de variables en Python (Rechaza strings si pides enteros, oculta contraseñas al enviar JSONs...)

pydantic-settings

Gestor avanzado de variables de entorno. Transforma las variables de `docker-compose` en variables tipadas y validadas seguras para Python.

python-dotenv

Librería para leer archivos estáticos `.env` físicos si no se usa Docker.

2. Frontend (React + Vite)

El frontend se erige sobre React 18 pero empaquetado por Vite, garantizando arranques en milisegundos en desarrollo.

frontend/ ├── src/ │ ├── assets/ # Imágenes estáticas y logo base │ ├── components/ # Componentes reusables (Layouts globales, Navbar, Footer, Sidebar) │ ├── pages/ # Vistas completas a las que el usuario accede vía URL ('Home', 'Login', 'Dashboard') │ ├── services/ # Lógica de llamadas a la API (api.ts base y servicios CRUD puntuales) │ ├── hooks/ # Hooks custom de React (estado y lógica compleja reutilizable) │ ├── App.tsx # Componente Orquestador de URLs (React Router) inyecta layouts │ ├── index.css # Inyección de instrucciones de TailwindCSS a todo el proyecto │ └── main.tsx # Punto de arranque donde React se inserta en el div '#root' ├── .prettierrc # Configuración de estandarizado automático de formato de código al guardar ├── eslint.config.js # Reglas de estandarización lógica de JS/TS (evita conflictos de equipo) ├── vite.config.ts # Configuración del servidor de desarrollo NodeJS/Vite └── package.json # Manifiesto de dependencias que se leen en 'npm install'

📚 Dependencias Clave (package.json)

react & react-dom

La librería fundamental del proyecto. Gestiona variables de estado y vuelve a pintar la web sin necesidad de recargarla (useEffect, useState).

react-router-dom

Habilita la navegación por pestañas invisibles. Cuando visitas "/admin", cambia los componentes en pantalla sin refrescar remotamente el navegador.

axios

Cliente HTTP moderno usado para hacer llamadas (GET, POST, PUT) a la API de FastAPI. Sustituye al fetch() nativo por mayor comodidad e interceptores.

framer-motion

Librería de animación fluida para React. Se encarga de las transiciones suaves, apariciones de la interfaz y efectos premium de la aplicación.

tailwindcss

Framework de clases CSS atómicas (dev-dependency). Permite darle estilo a tu app directamente en la etiqueta del componente (ej: className="bg-blue-500 rounded").

lucide-react

Colección de iconos modernos en vectores importables directamente como componentes estandar (ej: <Menu />).

typescript

Superconjunto de Javascript (dev-dependency). Añade un sistema de tipado estricto e inteligente para auto-completar el código y prevenir errores que solo saltearían en producción antes de guardarlos.

vite

Motor que sirve los archivos y empaqueta el producto final en /dist cuando está listo para despliegue en la vida real.
