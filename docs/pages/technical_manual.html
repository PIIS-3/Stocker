📘 Manual Técnico

Arquitectura de la API y conexión de servicios del Frontend con el backend FastAPI.

1. Arquitectura API (Backend)

1.1 Entrada HTTP

FastAPI inicializa la aplicación en backend/app/main.py e incluye el router principal de API.

1.2 Router principal

El router maestro en backend/app/api/router.py publica rutas bajo /api y delega por recurso (actualmente productos).

1.3 Endpoints de productos

Endpoints actuales:

  • GET /api/products/: listado paginado con skip y limit.
  • GET /api/products/{product_id}: detalle por ID con 404 si no existe.

La respuesta está validada por response_model con esquemas SQLModel/Pydantic.

1.4 Capa CRUD y DB Session

La lógica de consulta vive en backend/app/crud/products.py y usa Session de SQLModel definida en backend/app/database.py.

2. Conexión de servicios Frontend

2.1 Instancia Axios central

frontend/src/services/api.ts define base URL, timeout e interceptores. La URL se normaliza para apuntar siempre a /api.

2.2 Servicio de productos tipado

frontend/src/services/products.service.ts implementa:

  • getProducts(skip, limit)
  • getProductById(productId)

El contrato tipado de respuesta es ProductApi.

2.3 Consumo en vista administrativa

frontend/src/pages/admin/ProductsList.tsx consume el servicio con useEffect y maneja isLoading, errorMessage y render de tabla.

3. Flujo extremo a extremo

1) Vista React ejecuta productsService.getProducts() → 2) Axios llama /api/products/ → 3) FastAPI valida y delega a CRUD → 4) CRUD consulta SQLModel Session → 5) FastAPI responde JSON validado → 6) Frontend mapea y renderiza tabla.
