🛠️ Guía Paso a Paso: Implementación de un CRUD

Tutorial sobre cómo añadir de forma correcta, modular y escalable un nuevo recurso ("entidad") que abarque toda la estructura (Base de datos -> API -> Frontend).

En este ejemplo manual, añadiremos la nueva entidad ficticia Warehouse (Almacenes).

🤖 Fast-Track: Prompt para Inteligencia Artificial

Si estás usando un Agente IA (Cursor, Copilot, Cline...), no hace falta que leas toda la guía. Copia este bloque de texto, sustituye [NOMBRE_ENTIDAD] y envíaselo al chat:

Implementa un nuevo CRUD completo para la entidad "[NOMBRE_ENTIDAD]".

Utiliza los archivos de la entidad "Store" como tu estándar arquitectónico estricto:
- Modelos: backend/app/models/store.py (Usa Base, Create, Update, Response y el Modelo ORM puro).
- CRUD Pura: backend/app/crud/stores.py
- Endpoints: backend/app/api/endpoints/stores.py

REGLAS OBLIGATORIAS:
1. El modelo ORM de BD debe heredar de `TimestampMixin` (backend/app/models/mixins.py) para autogestionar created_at y updated_at de PostgreSQL. No insertes la hora manualmente.
2. Añade el nuevo modelo a backend/app/models/__init__.py
3. En el archivo CRUD, devuelve solo datos u objetos ORM (o None). NO importes ni lances HTTPException aquí.
4. En el archivo de endpoints, inyecta `Depends(get_db)`. Valida los conflictos de campos únicos (como nombres duplicados) usando el CRUD y lanzando `HTTP 409 Conflict`. Maneja los retornos `None` lanzando `HTTP 404 Not Found`. Usa siempre el `response_model`.
5. Los endpoints deben replicar EXACTAMENTE los comentarios visuales de separación (`# ── GET /entidad/ ──`) y definir los diccionarios `_404` y `_409` para insertarlos en el parámetro `responses=` de los decoradores de FastAPI (ej: `responses={**_404, **_409}`).
6. Crea siempre un endpoint adicional `GET /by-name/{name}` y ubícalo ANTES del endpoint `GET /{id}`.
7. Registra el router en backend/app/api/router.py.
8. Tipado fuerte en todo el código y usa model_validate() y model_dump(exclude_unset=True) para las actualizaciones.
9. El código debe estar excelentemente documentado y comentado en español, explicando el por qué de cada función y bloque, exactamente igual que está documentado el CRUD 'stores'.

Fase 1: Backend (Base de Datos y Modelos)

Paso 1.1 Crear Modelo SQLModel

Crea un archivo nuevo: backend/app/models/warehouse.py y define la herencia tripartita típica (Base, Create, Response, Model).

from typing import Optional
from datetime import datetime
from sqlmodel import SQLModel, Field
from .mixins import TimestampMixin

# Base que exportan las variables más repetidas
class WarehouseBase(SQLModel):
    name: str = Field(unique=True, index=True, min_length=1)
    location: str

# Los parámetros que el FrontEnd está forzado a mandar para crearla
class WarehouseCreate(WarehouseBase):
    pass

# Schema para actualización parcial (PATCH)
class WarehouseUpdate(SQLModel):
    name: Optional[str] = Field(default=None, min_length=1)
    location: Optional[str] = None

# La estructura Real de la Tabla de Base de Datos
class Warehouse(TimestampMixin, WarehouseBase, table=True):
    id_warehouse: Optional[int] = Field(default=None, primary_key=True)

# El cascarón de Respuesta hacia el Frontend (con timestamps)
class WarehouseResponse(WarehouseBase):
    id_warehouse: int
    created_at: Optional[datetime] = None
    updated_at: Optional[datetime] = None

Paso 1.2 Añadir el archivo al Índice

Abre backend/app/models/__init__.py e importa los nuevos modelos para que todo el proyecto y Alembic detecten los archivos.

from .warehouse import Warehouse, WarehouseBase, WarehouseCreate, WarehouseResponse

Fase 2: Backend (Operaciones y Rutas API)

Paso 2.1 Crear el archivo CRUD de Lógica de Negocio

Crea el archivo aislado backend/app/crud/warehouse.py y añade las mutaciones puras.

from sqlmodel import Session, select
from app.models.warehouse import Warehouse, WarehouseCreate, WarehouseUpdate

def get_warehouses(db: Session, skip: int = 0, limit: int = 100) -> list[Warehouse]:
    return db.exec(select(Warehouse).offset(skip).limit(limit)).all()

def get_warehouse_by_name(db: Session, warehouse_name: str) -> Warehouse | None:
    return db.exec(select(Warehouse).where(Warehouse.name == warehouse_name)).first()

def create_warehouse(db: Session, warehouse_in: WarehouseCreate) -> Warehouse:
    db_warehouse = Warehouse.model_validate(warehouse_in)
    db.add(db_warehouse)
    db.commit()
    db.refresh(db_warehouse)
    return db_warehouse

def update_warehouse(db: Session, warehouse_id: int, warehouse_in: WarehouseUpdate) -> Warehouse | None:
    db_warehouse = db.exec(select(Warehouse).where(Warehouse.id_warehouse == warehouse_id)).first()
    if db_warehouse is None:
        return None

    update_data = warehouse_in.model_dump(exclude_unset=True)
    for field, value in update_data.items():
        setattr(db_warehouse, field, value)

    db.add(db_warehouse)
    db.commit()
    db.refresh(db_warehouse)
    return db_warehouse

Paso 2.2 Publicar el Endpoint API

Crea backend/app/api/endpoints/warehouses.py. Esto generará las URL HTTP, ligándolas a la función CRUD creada anteriormente y forzando la validación del schema Response.

from fastapi import APIRouter, Depends, HTTPException, status
from sqlmodel import Session
from app.database import get_db
from app.crud import warehouse as crud
from app.models.warehouse import WarehouseResponse, WarehouseCreate, WarehouseUpdate

router = APIRouter()

@router.get("/", response_model=list[WarehouseResponse])
def read_warehouses(db: Session = Depends(get_db), skip: int = 0, limit: int = 100):
    return crud.get_warehouses(db, skip=skip, limit=limit)

@router.post("/", response_model=WarehouseResponse, status_code=status.HTTP_201_CREATED)
def create_warehouse(*, db: Session = Depends(get_db), warehouse_in: WarehouseCreate):
    if crud.get_warehouse_by_name(db, warehouse_name=warehouse_in.name):
        raise HTTPException(status_code=status.HTTP_409_CONFLICT, detail="Warehouse already exists")
    return crud.create_warehouse(db, warehouse_in)

@router.patch("/{warehouse_id}", response_model=WarehouseResponse)
def update_warehouse(warehouse_id: int, warehouse_in: WarehouseUpdate, db: Session = Depends(get_db)):
    updated = crud.update_warehouse(db, warehouse_id=warehouse_id, warehouse_in=warehouse_in)
    if updated is None:
        raise HTTPException(status_code=status.HTTP_404_NOT_FOUND, detail="Warehouse not found")
    return updated

Paso 2.3 Conectar al Enrutador Central

Para hacer que la URL cobre vida, importa tú nuevo endpoint en el router maestro ubicado en backend/app/api/router.py:

from app.api.endpoints import warehouses
# Añadir la siguiente línea:
api_router.include_router(warehouses.router, prefix="/warehouses", tags=["Warehouses"])

Fase 3: Frontend (Servicios y Vistas React)

Paso 3.1 Crear Servicio Frontend Typado

Crea frontend/src/services/warehouses.service.ts. Usa las interfaces de TypeScript idénticas al modelo que programaste en back, y usa la instancia Axios central.

import api from './api';

export interface Warehouse {
  id: number;
  name: string;
  location: string;
}

export const WarehouseService = {
  getAll: async () => {
    const response = await api.get<Warehouse[]>('/warehouses/');
    return response.data;
  },
  create: async (data: Omit<Warehouse, 'id'>) => {
    const response = await api.post<Warehouse>('/warehouses/', data);
    return response.data;
  }
};
