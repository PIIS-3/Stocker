🛠️ Guía Paso a Paso: Implementación de un CRUD

Tutorial sobre cómo añadir de forma correcta, modular y escalable un nuevo recurso ("entidad") que abarque toda la estructura (Base de datos -> API -> Frontend).

En este ejemplo, añadiremos la nueva entidad ficticia Warehouse (Almacenes).

Fase 1: Backend (Base de Datos y Modelos)

Paso 1.1 Crear Modelo SQLModel

Crea un archivo nuevo: backend/app/models/warehouse.py y define la herencia tripartita típica (Base, Create, Response, Model).

from sqlmodel import SQLModel, Field

# Base que exportan las variables más repetidas
class WarehouseBase(SQLModel):
    name: str
    location: str

# Los parámetros que el FrontEnd está forzado a mandar para crearla
class WarehouseCreate(WarehouseBase):
    pass

# La estructura Real de la Tabla de Base de Datos
class Warehouse(WarehouseBase, table=True):
    id: int | None = Field(default=None, primary_key=True)

# El cascarón de Respuesta hacia el Frontend (sin secretos)
class WarehouseResponse(WarehouseBase):
    id: int

Paso 1.2 Añadir el archivo al Índice

Abre backend/app/models/__init__.py e importa los nuevos modelos para que todo el proyecto y Alembic detecten los archivos.

from .warehouse import Warehouse, WarehouseBase, WarehouseCreate, WarehouseResponse

Paso 1.3 Generar Migración de Alembic

Ya que has modificado tu estructura de base de datos, ejecuta en la consola de docker:

docker exec stocker_api alembic revision --autogenerate -m "Add warehouse tables"
docker exec stocker_api alembic upgrade head

Fase 2: Backend (Operaciones y Rutas API)

Paso 2.1 Crear el archivo CRUD de Lógica de Negocio

Crea el archivo aislado backend/app/crud/warehouse.py y añade las mutaciones puras.

from sqlmodel import Session, select
from app.models.warehouse import Warehouse, WarehouseCreate

def get_warehouses(db: Session, skip: int = 0, limit: int = 100):
    return db.exec(select(Warehouse).offset(skip).limit(limit)).all()

def create_warehouse(db: Session, warehouse_in: WarehouseCreate):
    db_warehouse = Warehouse.model_validate(warehouse_in)
    db.add(db_warehouse)
    db.commit()
    db.refresh(db_warehouse)
    return db_warehouse

Paso 2.2 Publicar el Endpoint API

Crea backend/app/api/endpoints/warehouses.py. Esto generará las URL HTTP, ligándolas a la función CRUD creada anteriormente y forzando la validación del schema Response.

from fastapi import APIRouter, Depends
from sqlmodel import Session
from app.api import deps
from app.crud import warehouse as crud
from app.models.warehouse import WarehouseResponse, WarehouseCreate

router = APIRouter()

@router.get("/", response_model=list[WarehouseResponse])
def read_warehouses(db: Session = Depends(deps.get_db), skip: int = 0, limit: int = 100):
    return crud.get_warehouses(db, skip=skip, limit=limit)

@router.post("/", response_model=WarehouseResponse)
def create_warehouse(*, db: Session = Depends(deps.get_db), warehouse_in: WarehouseCreate):
    return crud.create_warehouse(db, warehouse_in)

Paso 2.3 Conectar al Enrutador Central

Para hacer que la URL cobre vida, importa tú nuevo endpoint en el router maestro ubicado en backend/app/api/router.py:

from app.api.endpoints import warehouses
# Añadir la siguiente línea:
api_router.include_router(warehouses.router, prefix="/warehouses", tags=["Warehouses"])

Fase 3: Frontend (Servicios y Vistas React)

Paso 3.1 Crear Servicio Frontend Typado

Crea frontend/src/services/warehouses.service.ts. Usa las interfaces de TypeScript idénticas al modelo que programaste en back, y usa la instancia Axios central.

import api from './api';

export interface Warehouse {
  id: number;
  name: string;
  location: string;
}

export const WarehouseService = {
  getAll: async () => {
    const response = await api.get<Warehouse[]>('/warehouses/');
    return response.data;
  },
  create: async (data: Omit<Warehouse, 'id'>) => {
    const response = await api.post<Warehouse>('/warehouses/', data);
    return response.data;
  }
};

Paso 3.2 Crear la Pagina Principal React

Crea un componente frontend/src/pages/admin/WarehousesList.tsx. Usa Hooks de React para llamar tu servicio en useEffect y renderiza el HTML correspondiente.

Paso 3.3 Registrar URL de navegación (React Router)

Inyecta la página que creaste arriba dentro de App.tsx en el grupo de rutas administrativas.

import WarehousesList from './pages/admin/WarehousesList';
// ... dentro de <Route path="/admin" ...>
<Route path="warehouses" element={<WarehousesList />} />

Paso 3.4 Añadir Botón de Navegación Visual en Sidebar

Por último, edita el componente Navbar/Sidebar administrativo para añadir un enlance HTML hacia tu recién creada ruta /admin/warehouses.
