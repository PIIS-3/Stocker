🚀 Guía de Operaciones (Getting Started)

Aprende a levantar el proyecto, apagarlo, limpiarlo e inyectar datos falsos.

Aviso Importante: La terminal debe estar abierta en la carpeta /docker o debes añadir cd docker antes de ejecutar estos comandos o fallarán.

1. Encender el Proyecto (Arranque Limpio)

Este comando construye las imágenes (Frontend en Node 25, Backend en Python 3.12 y Database en Postgres 16) y lanza los tres contenedores en segundo plano -d.

docker compose up --build -d

Una vez completado, las herramientas estarán en:

  • Frontend React: http://localhost:5173
  • Backend API: http://localhost:8000 (Documentación interactiva en /docs)

2. Inyectar datos de prueba (Seed Database)

Como Postgres arranca en limpio, FastAPI usará Alembic para crear las tablas vacías. Una vez estén creadas (10 segundos tras encender el servidor), debes ejecutar la siembra (Seed) de forma manual si quieres tener Categorías, Roles, Tiendas y un Usuario Administrador pre-cargado. Se hace inyectando directamente el script mediante un conducto (pipe) hacia el motor interactivo psql de la base de datos.

docker exec -i stocker_db psql -U postgres -d stocker < ../database/seed.sql

Si la inserción va bien, la consola te mostrará un listado de confirmaciones INSERT 0 3.

3. Realizar cambios en la Base de Datos (Migraciones)

Si alteras el archivo models.py de Python para añadir o quitar columnas, debes ordenarle a Alembic que compare el modelo nuevo con el antiguo de Postgres y genere el script de actualización.

  1. Genera la captura autogenerada:
    docker exec stocker_api alembic revision --autogenerate -m "He añadido una columna de Stock"
  2. Aplica la actualización a la base de datos real:
    docker exec stocker_api alembic upgrade head

4. Apagar y Limpiar

Si quieres apagarlo todo temporalmente para conservar recursos y disco:

docker compose down

Si notas fallos extraños y quieres **borrar todo el progreso (Resetear de Fábrica)**, eliminando la base de datos, los datos guardados en el disco virtual de Docker y las imágenes construidas, usa:

docker compose down -v --rmi all --remove-orphans
