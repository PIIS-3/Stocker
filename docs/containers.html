Stocker - Arquitectura Docker

Documentación técnica de la orquestación de contenedores en desarrollo local.

Resumen del Sistema

El proyecto Stocker está separado en una arquitectura de microservicios de tres capas orquestados mediante docker-compose. Esto permite que el Frontend, la API Backend y la Base de Datos corran en entornos aislados pero comunicados a través de una red virtual interna de Docker.

Orquestador Principal

  • docker/docker-compose.yml

    El archivo maestro que define la orquestación. Levanta los 3 contenedores (stocker_web, stocker_api, stocker_db), crea la red interna, mapea los volúmenes para el Hot-Reloading de código y expone los puertos locales necesarios (5173, 8000, 5432).

  • docker/.env.example

    Plantilla de las variables de entorno operativas necesarias para inyectar configuración tanto a la base de datos (credenciales) como a la API y el Frontend.

Containers (Servicios)

📁 Database stocker_db (postgres:16-alpine)

Contenedor de almacenamiento persistente corriendo PostgreSQL V16. Ahora se arranca completamente en blanco para entregar el control de arquitecturas a código Python (Alembic). Los datos de validación inicial deben cargarse de forma manual con seed.sql tras el arranque.

Archivos de Configuración e Inicialización:

  • docker-compose.yml (Config)

    Define el entorno limpio, mapeando el volumen etiquetado postgres_data para que, aunque se destruya el contenedor, los datos persistan de forma segura en disco físico local.

⚙️ Backend API stocker_api (python:3.12-slim)

Servidor RESTful moderno escrito en FastAPI usando Uvicorn y conectado a la BD PostgreSQL a través de la arquitectura unificada de SQLModel.

Archivos de Construcción y Migraciones:

  • backend/Dockerfile

    Script de construcción para la moderna imagen de Python 3.12. Instala dependencias del sistema operativo nativas (como gcc y libpq-dev) para compilar los drivers internos de PostgreSQL y habilita el pip local.

  • alembic.ini & alembic/

    Gestor de migraciones autónomo. Docker ejecuta el mapeo en vivo alembic upgrade head antes de arrancar la API. Alembic se encarga de crear las tablas de SQLModel sobre Postgres la primera vez que arranca.

💻 Frontend stocker_web (node:25-alpine)

Aplicación SPA generada en React con TypeScript gestionada dinámicamente in-memory por ViteJS bajo Node 25.

Archivos de Construcción:

  • frontend/Dockerfile

    Imagen Node optimizada. Lanza npm install --legacy-peer-deps en Build, y levanta el servidor de ViteJS exponiéndolo fuera del contenedor permitiendo Hot-Reloading instantáneo.

Comandos de Interacción Centrales

Para encender o recargar todo el sistema usando estos archivos de configuración, desde la raíz del proyecto ejecutamos:

cd docker
docker compose down          # Apaga los contendores actuales si existiesen
docker compose up --build -d # (Re)Compila usando los Dockerfiles listados e inicia
